fn set_align_widget<P: IsA<Widget>>(&self, align_widget: Option<&P>)[src]

Sets the crate::Widget to use to line the menu with when popped up. Note that the align_widget must contain the crate::MenuButton itself.

Setting it to None means that the menu will be aligned with the button itself.

Note that this property is only used with menus currently, and not for popovers.

align_widget

a crate::Widget

fn set_direction(&self, direction: ArrowType)[src]

Sets the direction in which the popup will be popped up, as well as changing the arrow’s direction. The child will not be changed to an arrow if it was customized.

If the does not fit in the available space in the given direction, GTK+ will its best to keep it inside the screen and fully visible.

If you pass crate::ArrowType::None for a direction, the popup will behave as if you passed crate::ArrowType::Down (although you won’t see any arrows).

direction

a crate::ArrowType

fn set_menu_model<P: IsA<MenuModel>>(&self, menu_model: Option<&P>)[src]

Sets the crate::gio::MenuModel from which the popup will be constructed, or None to dissociate any existing menu model and disable the button.

Depending on the value of [crate::MenuButton:use-popover], either a crate::Menu will be created with [crate::Menu::new_from_model()], or a crate::Popover with [crate::Popover::new_from_model()]. In either case, actions will be connected as documented for these functions.

If [crate::MenuButton:popup] or [crate::MenuButton:popover] are already set, those widgets are dissociated from the self, and those properties are set to None.

menu_model

a crate::gio::MenuModel, or None to unset and disable the button

fn set_popover<P: IsA<Widget>>(&self, popover: Option<&P>)[src]

Sets the crate::Popover that will be popped up when the self is clicked, or None to dissociate any existing popover and disable the button.

If [crate::MenuButton:menu-model] or [crate::MenuButton:popup] are set, those objects are dissociated from the self, and those properties are set to None.

popover

a crate::Popover, or None to unset and disable the button

fn set_popup<P: IsA<Widget>>(&self, menu: Option<&P>)[src]

Sets the crate::Menu that will be popped up when the self is clicked, or None to dissociate any existing menu and disable the button.

If [crate::MenuButton:menu-model] or [crate::MenuButton:popover] are set, those objects are dissociated from the self, and those properties are set to None.

menu

fn set_use_popover(&self, use_popover: bool)[src]

Sets whether to construct a crate::Popover instead of crate::Menu when Self::set_menu_model() is called. Note that this property is only consulted when a new menu model is set.
